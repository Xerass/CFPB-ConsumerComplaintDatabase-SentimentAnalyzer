# CFPB Consumer Complaint Classifier

A multi-class text classification pipeline that automatically categorizes consumer financial complaints into product categories using data from the U.S. Consumer Financial Protection Bureau (CFPB).

## Problem

The CFPB receives a constant stream of consumer complaints that must be routed to the correct specialist team (mortgage, credit reporting, debt collection, etc.). Manual triage is slow, expensive, and inconsistent. This project builds a classifier that predicts the correct product category from a complaint's narrative text and metadata, enabling automated routing.

## Dataset

**Source:** [CFPB Consumer Complaint Database](https://www.consumerfinance.gov/data-research/consumer-complaints/) (public domain)

- **30,000 complaints** balanced across 6 classes (5,000 each)
- Sampled with `random_state=42` for reproducibility

| Class | Description |
|---|---|
| Bank Account | Checking/savings account issues |
| Credit Card | Credit card and prepaid card complaints |
| Credit Reporting | Credit report disputes, bureau complaints |
| Debt Collection | Third-party debt collection issues |
| Mortgage | Home loans, escrow, foreclosure |
| Other | Out-of-scope catch-all (student loans, vehicle loans, money transfers, etc.) |

The "Other" class is built via stratified sampling across 7 discarded CFPB categories, giving the classifier an explicit out-of-scope bucket rather than forcing every complaint into the top 5.

## Quick Start

### 1. Install dependencies

```bash
pip install pandas numpy scipy scikit-learn matplotlib seaborn nltk wordcloud requests tqdm
```

### 2. Get the dataset

```bash
python data_download.py
```

This downloads the full CFPB complaints CSV (~400-500 MB compressed), normalizes duplicate labels, and samples a balanced 30k-row dataset.

> **Note:** The CFPB endpoint may return a 403 error due to bot detection. If the download fails, manually download the ZIP from [https://files.consumerfinance.gov/ccdb/complaints.csv.zip](https://files.consumerfinance.gov/ccdb/complaints.csv.zip) and place `complaints.csv.zip` in the project root. The script detects existing files and skips the download.

### 3. Run the pipeline

Open `pipeline.ipynb` and run all cells. The notebook handles everything end-to-end: text cleaning, feature engineering, model training, evaluation, and visualization.

An interactive prediction cell at the end lets you paste your own complaint text and get a live classification.

## Pipeline Overview

### Text Preprocessing
- PII scrubbing (CFPB redaction artifacts like `XXXX`, `XX/XX/XXXX`, `$XXXX`)
- Lowercasing, non-alphabetic removal, whitespace normalization
- Brand-name stopword filtering to prevent overfitting on company names

### Feature Engineering
- **Word TF-IDF** (1-2 grams, 30k features) with sublinear TF and NLTK stopwords
- **Character TF-IDF** (3-5 grams, 30k features) for misspellings and morphological variants
- **One-hot encoded categoricals** (surviving a dominance audit that drops leaky/low-signal columns)
- **Standardized numerics** (narrative length, word count)
- **Pair-discriminator phrases** (31 regex-matched binary features targeting known confusion pairs)

### Models

Three classifiers spanning probabilistic, max-margin, and connectionist paradigms:

| Model | Features | Accuracy | Macro F1 | Acc (Accepted) |
|---|---|---|---|---|
| **Linear SVM** | Combined | **0.8227** | **0.8216** | **0.8617** |
| Neural Network (MLP) | Combined | 0.8125 | 0.8140 | 0.8540 |
| Multinomial Naive Bayes | Text only | 0.7907 | 0.7872 | 0.8302 |

### Out-of-Scope Rejection (Tau Threshold)

Each model has a calibrated confidence threshold (tau) that routes low-confidence predictions to "Out of Scope" instead of forcing a class pick. Tau is set on a held-out validation split to retain ~95% of correctly classified samples.

| Model | Tau | Coverage | Selective Gain |
|---|---|---|---|
| Linear SVM | 0.316 | 89.7% | +3.9% |
| MLP | 0.496 | 90.4% | +4.2% |
| Naive Bayes | 0.681 | 90.8% | +4.0% |

Coverage = fraction of test rows the model commits to (the rest are rejected as uncertain). Accuracy on accepted predictions is consistently higher than raw accuracy, which is the value the rejection layer adds.

### Best Model: Linear SVM

Per-class breakdown (on 6,000 test samples):

| Class | Precision | Recall | F1 |
|---|---|---|---|
| Bank Account | 0.829 | 0.886 | 0.857 |
| Credit Card | 0.810 | 0.803 | 0.807 |
| Credit Reporting | 0.818 | 0.829 | 0.824 |
| Debt Collection | 0.764 | 0.804 | 0.784 |
| Mortgage | 0.923 | 0.928 | 0.926 |
| Other | 0.789 | 0.686 | 0.734 |

Mortgage is the easiest class to separate (F1 0.926). The main confusion pairs are Debt Collection vs. Credit Reporting and Credit Card vs. Bank Account, which share significant vocabulary overlap.

## Data Download Details

`data_download.py` handles the full dataset build:

1. **Downloads** the CFPB complaints ZIP (~400-500 MB) with progress bar (skips if file exists)
2. **Extracts** the CSV (~4 GB uncompressed)
3. **Normalizes labels** -- the CFPB renamed several product categories in 2022 but both old and new names persist in the data (e.g., "Credit reporting or other personal consumer reports" and "Credit reporting, credit repair services, or other personal consumer reports" both map to "Credit Reporting")
4. **Counts** rows per normalized class (streaming, memory-efficient)
5. **Samples** 5,000 rows per top-5 class + 5,000 stratified "Other" rows from all remaining classes
6. **Outputs** `cfpb_with_other_30k.csv` and a build summary

### Configuration

Key parameters in `data_download.py`:

| Parameter | Default | Description |
|---|---|---|
| `SAMPLES_PER_CLASS` | 5,000 | Rows per class in the output |
| `NUM_CLASSES` | 5 | Top N product categories to keep |
| `INCLUDE_OTHER_CLASS` | True | Build a 6th "Other" class for OOD |
| `RANDOM_SEED` | 42 | Reproducibility seed |
